## SC-001: 지역별 주택 거래 데이터 검색

| 항목 | 내용 |
|------|------|
| 시나리오명 | 지역별 주택 거래 검색 |
| 액터 | 비회원, 회원 |
| 사전 조건 | 메인 페이지 또는 검색 페이지에 접속한 상태 |
| 관련 요구사항 | REQ-HOUSE-002 |

**주요 흐름:**
1. 사용자가 시도·시군구·읍면동을 선택한다.
2. 검색 버튼을 클릭한다.
3. 시스템이 선택 조건에 해당하는 주택 거래 목록을 반환한다.
4. 사용자가 목록에서 거래 건수, 가격, 주택명을 확인한다.

**대안 흐름:**
- 해당 조건의 데이터가 없으면 "검색 결과가 없습니다" 메시지를 표시한다.
- 행정구역 코드가 유효하지 않으면 오류 메시지를 표시한다.

## SC-002: 매물 상세 정보 조회

| 항목 | 내용 |
|------|------|
| 시나리오명 | 매물 상세 정보 조회 |
| 액터 | 비회원, 회원 |
| 사전 조건 | 주택 거래 검색 결과 목록이 표시된 상태 |
| 관련 요구사항 | REQ-HOUSE-003 |

**주요 흐름:**
1. 사용자가 목록에서 특정 주택을 선택한다.
2. 시스템이 해당 주택의 거래 이력, 면적, 층, 위치 정보를 상세 페이지에 표시한다.
3. 지도에 매물 위치가 핀으로 표시된다.

**대안 흐름:**
- 해당 주택 ID가 존재하지 않으면 404 메시지를 표시한다.

## SC-003: 관심 지역 등록 및 관리

| 항목 | 내용 |
|------|------|
| 시나리오명 | 관심 지역 등록 및 관리 |
| 액터 | 회원 |
| 사전 조건 | 로그인 상태 |
| 관련 요구사항 | REQ-FAVORITE-001, REQ-FAVORITE-002, REQ-FAVORITE-003 |

**주요 흐름:**
1. 회원이 검색 결과 또는 관심 지역 관리 페이지에서 특정 행정구역을 선택한다.
2. "관심 지역 추가" 버튼을 클릭한다.
3. 시스템이 해당 구역을 관심 지역으로 저장한다.
4. 관심 지역 관리 페이지에서 저장된 목록을 확인한다.
5. 특정 관심 지역의 삭제 버튼을 클릭해 목록에서 제거한다.

**대안 흐름:**
- 이미 등록된 관심 지역을 다시 등록하려 하면 중복 오류 메시지를 표시한다.
- 로그인하지 않은 상태에서 등록을 시도하면 로그인 페이지로 이동한다.

## SC-004: 주변 상권 정보 조회

| 항목 | 내용 |
|------|------|
| 시나리오명 | 주변 상권 정보 조회 |
| 액터 | 비회원, 회원 |
| 사전 조건 | 매물 상세 페이지 또는 상권 정보 페이지 접근 |
| 관련 요구사항 | REQ-COMMERCIAL-001, REQ-COMMERCIAL-002 |

**주요 흐름:**
1. 사용자가 매물 상세 페이지에서 "주변 상권" 탭을 선택한다.
2. 시스템이 매물 위치 주변 상업 시설 목록을 반환한다.
3. 사용자가 업종 필터(음식점, 편의점, 병원 등)를 적용해 원하는 유형만 확인한다.

**대안 흐름:**
- 주변 상권 데이터가 없으면 "주변 상권 정보가 없습니다" 메시지를 표시한다.
- 외부 API 응답 오류 시 "현재 상권 정보를 불러올 수 없습니다" 메시지를 표시한다.

## SC-006: 매물에서 목적지까지 경로 탐색

| 항목 | 내용 |
|------|------|
| 시나리오명 | A* 경로 탐색 |
| 액터 | 회원 |
| 사전 조건 | 로그인 상태. 매물 상세 페이지 접근 |
| 관련 요구사항 | REQ-ROUTE-001 |

**주요 흐름:**
1. 회원이 매물 상세 페이지의 경로 탐색 패널을 열고 목적지를 입력한다.
2. "경로 탐색" 버튼을 클릭한다.
3. 시스템이 매물 위치(출발지)와 입력 목적지를 기반으로 A* 알고리즘으로 최적 경로를 계산한다.
4. 계산 결과(총 거리, 예상 시간, 경로 노드 목록)가 반환된다.
5. 지도에 경로가 선으로 표시된다.

**대안 흐름:**
- 목적지가 그래프 범위 밖이면 "탐색 가능 범위를 벗어났습니다" 메시지를 표시한다.
- 경로가 존재하지 않으면 "경로를 찾을 수 없습니다" 메시지를 표시한다.
- 로그인하지 않은 상태에서 시도하면 로그인 유도 메시지를 표시한다.

## SC-007: 회원 정보 관리

| 항목 | 내용 |
|------|------|
| 시나리오명 | 회원 정보 조회 및 수정 |
| 액터 | 회원 |
| 사전 조건 | 로그인 상태. 마이페이지 접근 |
| 관련 요구사항 | REQ-MEMBER-002, REQ-MEMBER-003, REQ-MEMBER-004 |

**주요 흐름:**
1. 회원이 내비게이션의 마이페이지 버튼을 클릭한다.
2. 현재 회원 정보(이름, 이메일, 가입일)가 표시된다.
3. 수정 버튼을 눌러 이름 또는 비밀번호를 변경한다.
4. 저장 버튼을 클릭하면 수정 사항이 반영된다.

**대안 흐름:**
- 비밀번호 변경 시 현재 비밀번호 확인이 실패하면 오류 메시지를 표시한다.
- 회원 탈퇴를 선택하면 확인 다이얼로그를 표시하고, 확인 시 계정을 삭제한다.

## SC-008: 공지사항 관리

| 항목 | 내용 |
|------|------|
| 시나리오명 | 공지사항 조회 및 작성 |
| 액터 | 비회원(조회), 관리자(작성·수정·삭제) |
| 사전 조건 | 공지사항 목록 페이지 접근 |
| 관련 요구사항 | REQ-NOTICE-001, REQ-NOTICE-002, REQ-NOTICE-003, REQ-NOTICE-004, REQ-NOTICE-005 |

**주요 흐름:**
1. 모든 사용자가 공지사항 목록 페이지에서 공지 목록을 조회한다.
2. 특정 공지를 클릭해 상세 내용을 확인한다.
3. 관리자는 "공지 작성" 버튼으로 새 공지를 등록한다.
4. 기존 공지를 선택해 수정 또는 삭제할 수 있다.

**대안 흐름:**
- 권한이 없는 사용자가 작성·수정·삭제를 시도하면 권한 오류 메시지를 표시한다.
