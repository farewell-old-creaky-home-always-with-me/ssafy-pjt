## 프론트엔드 구성 원칙

- 현재 화면 흐름은 HTML/CSS/JavaScript 기준으로 설계한다.
- 백엔드는 REST API만 제공하고 서버 측 페이지 렌더링에 의존하지 않는다.
- 향후 Vue.js 전환 시에도 동일한 화면 ID와 흐름을 Vue Router 구조로 옮길 수 있도록 설계한다.
- 관리자 배치 운영 화면은 향후 확장 범위이며 현재 사용자 화면 흐름에는 포함하지 않는다.

## 비회원 사용자 흐름

비회원은 검색, 조회, 공지사항 열람만 가능하다. 회원 전용 기능에 접근하면 로그인 페이지로 이동한다.

```mermaid
flowchart LR
    시작([서비스 접속]) --> 메인[메인 페이지]
    메인 --> 검색[주택 검색]
    검색 --> 결과[검색 결과]
    결과 --> 상세[매물 상세]
    상세 --> 상권정보[상권 정보 탭]
    상세 --> 환경정보[환경 정보 탭]
    상세 -->|경로 탐색 시도| 로그인유도[로그인 유도 메시지]
    로그인유도 --> 로그인[로그인 페이지]
    메인 --> 공지목록[공지사항 목록]
    공지목록 --> 공지상세[공지사항 상세]
```

```mermaid
flowchart TD
```

## A* 경로 탐색 흐름

```mermaid
flowchart TD
    상세페이지[매물 상세 페이지] --> 인증확인{로그인 상태?}
    인증확인 -->|아니오| 로그인유도[로그인 유도]
    인증확인 -->|예| 패널열기[경로 탐색 패널 열기]
    패널열기 --> 목적지입력[목적지 입력\n위도·경도 또는 주소]
    목적지입력 --> 탐색실행[경로 탐색 버튼 클릭]
    탐색실행 --> API호출[GET /api/routes 호출]
    API호출 --> 탐색결과{탐색 성공?}
    탐색결과 -->|성공| 결과표시[총 거리·예상 시간·경로 표시\n지도에 경로 렌더링]
    탐색결과 -->|범위 초과| 범위오류[탐색 가능 범위 초과 메시지]
    탐색결과 -->|경로 없음| 경로없음[경로를 찾을 수 없음 메시지]
```

## 공지사항 흐름

```mermaid
flowchart TD
    공지목록[공지사항 목록 페이지] --> 공지선택[공지 항목 클릭]
    공지선택 --> 공지상세[공지사항 상세 페이지]
    공지목록 --> 권한확인{관리자?}
    권한확인 -->|예| 작성버튼[공지 작성 버튼]
    작성버튼 --> 공지작성폼[공지사항 작성 페이지]
    공지작성폼 --> 저장[저장 → 목록으로 이동]
    공지상세 --> 수정권한{관리자?}
    수정권한 -->|예| 수정버튼[수정 버튼]
    수정버튼 --> 공지수정폼[공지사항 수정 페이지]
    공지수정폼 --> 저장
    공지상세 --> 삭제권한{관리자?}
    삭제권한 -->|예| 삭제버튼[삭제 버튼]
    삭제버튼 --> 삭제확인[확인 다이얼로그]
    삭제확인 --> 삭제완료[삭제 → 목록으로 이동]
```
