## 비기능 요구사항 목록

### 사용성 (Usability)

| ID | 분류 | 설명 | 수용 기준 | 관련 기능 요구사항 |
|----|------|------|-----------|-------------------|
| NFR-USE-001 | 사용성 | 신규 사용자가 별도 안내 없이 주택 검색 기능을 사용할 수 있어야 한다 | 검색 입력 → 결과 확인까지 3단계 이내에 완료 가능 | REQ-HOUSE-002 |
| NFR-USE-002 | 사용성 | 로그인·가입 화면은 사용자가 필드 오류를 즉시 인지할 수 있어야 한다 | 폼 제출 실패 시 오류 메시지가 해당 필드 근처에 표시됨 | REQ-AUTH-001, REQ-MEMBER-001 |
| NFR-USE-003 | 사용성 | 모든 주요 기능은 화면 로딩 후 별도 스크롤 없이 핵심 UI 요소를 확인할 수 있어야 한다 | 1280×800 해상도 기준 Above-the-fold 영역에 핵심 요소 노출 | 전체 |

### 보안 (Security)

| ID | 분류 | 설명 | 수용 기준 | 관련 기능 요구사항 |
|----|------|------|-----------|-------------------|
| NFR-SEC-001 | 보안 | 비밀번호는 평문으로 저장하지 않는다 | 비밀번호는 해시(bcrypt 등) 처리 후 저장 | REQ-MEMBER-001, REQ-AUTH-001 |
| NFR-SEC-002 | 보안 | 로그인하지 않은 사용자는 회원 전용 기능에 접근할 수 없다 | 미인증 요청 시 HTTP 401 반환 | REQ-FAVORITE-001, REQ-ROUTE-001 |
| NFR-SEC-003 | 보안 | SQL Injection 취약점이 없어야 한다 | MyBatis PreparedStatement 사용으로 동적 쿼리 입력값 이스케이프 | 전체 데이터 조회 API |
| NFR-SEC-004 | 보안 | XSS 취약점이 없어야 한다 | 사용자 입력값을 HTML 렌더링 시 이스케이프 처리 | REQ-NOTICE-003, REQ-NOTICE-004 |

### 유지보수성 (Maintainability)

| ID | 분류 | 설명 | 수용 기준 | 관련 기능 요구사항 |
|----|------|------|-----------|-------------------|
| NFR-MAIN-001 | 유지보수성 | 백엔드 코드는 컨트롤러·서비스·매퍼 레이어를 명확히 분리한다 | 각 클래스가 단일 책임 원칙을 따름 | 전체 |
| NFR-MAIN-002 | 유지보수성 | API 오류 응답 형식이 전체 서비스에서 일관되어야 한다 | 모든 에러 응답이 공통 에러 응답 포맷 준수 | 전체 API |
| NFR-MAIN-003 | 유지보수성 | SQL 쿼리는 MyBatis XML Mapper에 정의하고 Java 코드에 인라인으로 작성하지 않는다 | Mapper XML 파일 외부에 SQL 문자열 없음 | 전체 |
