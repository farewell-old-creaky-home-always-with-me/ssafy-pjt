## 요구사항 표

### 주택 거래 정보 (HOUSE)

| ID | 분류 | 요구사항명 | 설명 | 우선순위 | 관련 화면 | 관련 API |
|----|------|-----------|------|----------|-----------|----------|
| REQ-HOUSE-001 | 데이터 수집 | 주택 거래 데이터 수집 | 관리자 API가 `houseDealCollectJob`을 실행하고, Spring Batch가 국토교통부 API 데이터를 수집·정규화·저장한다 | 필수 | 없음 (배치) | POST /api/admin/batch/house-deals |
| REQ-HOUSE-002 | 검색 | 주택 거래 검색 | 시도·시군구·읍면동 코드 조건으로 주택 거래 목록을 조회한다 | 필수 | 주택 검색 결과 페이지 | GET /api/houses |
| REQ-HOUSE-003 | 상세 조회 | 주택 상세 조회 | 특정 주택의 거래 이력, 위치, 면적, 층, 건축연도 등 상세 정보를 조회한다 | 필수 | 매물 상세 페이지 | GET /api/houses/{houseId} |
| REQ-HOUSE-004 | 검색 | 거래 유형 필터 | 아파트·다세대 유형별로 검색 결과를 필터링한다 | 추가 | 주택 검색 결과 페이지 | GET /api/houses |
| REQ-HOUSE-005 | 검색 | 거래 금액 범위 필터 | 최소·최대 거래금액 범위로 검색 결과를 필터링한다 | 추가 | 주택 검색 결과 페이지 | GET /api/houses |

주택 거래 데이터 수집은 일반 요청-응답 Service 로직이 아니라 Spring Batch Job으로 처리한다. 관리자 API는 Job 실행과 상태 확인의 진입점 역할만 담당한다.

## 우선순위 정의

| 우선순위 | 설명 |
|----------|------|
| 필수 | 서비스 핵심 기능. 반드시 구현해야 한다 |
| 추가 | 서비스 품질을 높이는 기능. 필수 완료 후 구현한다 |
| 심화 | 알고리즘 등 고도화 기능. 추가 완료 후 구현한다 |
