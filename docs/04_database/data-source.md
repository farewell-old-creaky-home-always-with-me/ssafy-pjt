## 데이터 소스 목록

### DS-001: 국토교통부 아파트 실거래가 API

| 항목 | 내용 |
|------|------|
| 출처 | 공공 데이터 포털 (data.go.kr) |
| 목적 | 아파트 매매 실거래 데이터 수집 |
| 수집 방법 | REST API 호출 (공공 데이터 포털 API 키 필요) |
| 업데이트 주기 | 월 단위 (공공 데이터 포털 기준) |
| 저장 테이블 | `house`, `house_deal` |

**주요 필드:**

| 공공 API 필드명 | 설명 | DB 컬럼 매핑 |
|----------------|------|-------------|
| 아파트 | 아파트 단지명 | house.apt_name |
| 법정동코드 | 행정구역 코드 | house.region_code |
| 지번 | 지번 주소 | house.jibun |
| 거래금액 | 실거래 금액 (만 원) | house_deal.deal_amount |
| 거래연도/거래월/거래일 | 거래 일자 구성 값 | house_deal.deal_date |
| 전용면적 | 전용 면적 (㎡) | house_deal.area |
| 층 | 층수 | house_deal.floor |
| 건축연도 | 건축 연도 | house.build_year |
| 거래구분 | 아파트 실거래 수집 시 기본값 | house_deal.deal_type = `매매` |

**데이터 전처리:**
- 금액 필드의 쉼표 제거 후 정수 변환
- 아파트 실거래 수집 건은 `house_deal.deal_type`을 `매매`로 저장
- 매매 수집 건은 `deposit_amount`, `monthly_rent`를 모두 `NULL`로 저장해 DB 제약을 만족시킨다.
- 거래연도·월·일을 DATE 형식으로 조합
- 중복 단지 처리: `region_code + apt_name + jibun` 조합으로 중복 확인

### DS-002: 국토교통부 다세대 실거래가 API

| 항목 | 내용 |
|------|------|
| 출처 | 공공 데이터 포털 (data.go.kr) |
| 목적 | 다세대·연립주택 매매 실거래 데이터 수집 |
| 수집 방법 | REST API 호출 |
| 업데이트 주기 | 월 단위 |
| 저장 테이블 | `house`, `house_deal` |

DS-001과 동일한 필드 구조를 사용한다. `house.house_type` 컬럼으로 아파트/다세대를 구분하고, 매매 수집 건은 `house_deal.deal_type`을 `매매`로 저장하며 `deposit_amount`, `monthly_rent`는 `NULL`로 저장한다.

### DS-003: VWorld 행정구역 API

| 항목 | 내용 |
|------|------|
| 출처 | 국토지리정보원 VWorld (vworld.kr) |
| 목적 | 행정구역 코드 및 명칭 데이터 구축 |
| 수집 방법 | API 또는 정적 파일 다운로드 |
| 업데이트 주기 | 연 1회 (행정구역 변경 시 갱신) |
| 저장 테이블 | `region_code` |

**주요 필드:**

| 필드 | 설명 |
|------|------|
| 법정동코드 | 행정구역 식별 코드 (10자리) |
| 시도명 | 시도 명칭 |
| 시군구명 | 시군구 명칭 |
| 읍면동명 | 읍면동 명칭 |
| 폐지여부 | 현재 유효 여부 |

**데이터 전처리:**
- 폐지 행정구역 제외
- 코드 길이 정규화 (10자리)

|------|------|
| 출처 | 미정 (소상공인 상권 정보 시스템 또는 공공 데이터 포털) |

**예상 필드:**
- 상호명, 업종 대분류·중분류·소분류, 위도, 경도, 주소

**데이터 품질 고려사항:**
- 폐업 시설 데이터 포함 가능. 최신성 확인 필요

### DS-005: 서울 열린데이터 광장 환경 점검 API

| 항목 | 내용 |
|------|------|
| 출처 | 서울 열린데이터 광장 (data.seoul.go.kr) |
| 목적 | 서울 지역 환경 점검 데이터 조회 |
| 수집 방법 | REST API 호출 (서울시 API 키 필요) |
| 업데이트 주기 | 미정 (API 데이터 갱신 주기에 따름) |
| 저장 테이블 | `environment_info` |
| 서비스 범위 제한 | 서울 지역에만 환경 정보 제공 가능 |

**예상 필드:**
- 점검 항목명, 측정 수치, 단위, 측정일, 위도, 경도

**데이터 품질 고려사항:**
- API 응답 데이터가 실시간이 아닐 수 있음. 측정일 기준으로 표시
- 서울 외 지역은 데이터 없음 처리 필요
